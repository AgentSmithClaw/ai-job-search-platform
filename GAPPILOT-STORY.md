# GapPilot — 从方法论到产品的完整故事

## 为什么做

求职时最花时间的不是投简历，而是：
- 判断"这个岗位我该不该投"
- 分析"我的简历和岗位差在哪里"
- 决定"简历该怎么改才能匹配"

这套判断过程高度依赖个人经验，而且每次换岗位都要重新做一遍。我在自己的求职过程中，把这套方法论沉淀成了系统化的分析流程。

## 怎么做的

**第一阶段：方法论沉淀（GapPilot Playbook）**

先用 4399 AI 研究助理作为目标岗位，完整走了一遍：
- JD 拆解 → 能力要求提取
- 简历证据映射 → 找到优势和差距
- 学习路径规划 → 弥补差距
- 面试准备 → 结构化表达
- 投递管理 → 跟踪与复盘

这套方法论沉淀为 [GapPilot Playbook](https://github.com/AgentSmithClaw/ai-job-search-playbook)，包含 17 个模块、100+ 份文档。

**第二阶段：产品化（GapPilot Platform）**

发现方法论的核心判断逻辑可以被自动化，于是做了 GapPilot Platform：
- 上传简历 → 自动解析（PDF/Word/OCR）
- 粘贴 JD → AI 提取能力要求
- 自动分析 → 生成匹配报告、差距项、改写建议
- 投递管理 → 跟踪求职进度

## 做了什么

| 维度 | Playbook | Platform |
|------|----------|----------|
| 形态 | Markdown 文档体系 | FastAPI + Vanilla JS 全栈应用 |
| 核心价值 | 方法论可复用 | 分析过程可自动化 |
| 规模 | 17 模块、100+ 文档 | 后端 ~3100 行 + 前端 ~2766 行 + 49 测试 |
| 状态 | 已完成 | 可运行 |

## 技术亮点

1. **简历解析多层降级**：PDF 文字提取 → OCR 扫描件兜底
2. **LLM 模型路由**：支持 OpenAI / MiniMax 等多模型切换
3. **Mock + Stripe 双模式支付**：未配置 API Key 也能完整体验
4. **本地可信度校验**：LLM 输出二次校验，减少幻觉
5. **49 个 pytest 测试**：后端核心逻辑全覆盖
